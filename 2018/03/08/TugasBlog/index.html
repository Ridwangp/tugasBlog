Panduan Singkat Penggunaan git

Git untuk Pemula

Instalasi

Download git di https://gitforwindows.org/ untuk pengguna windows

Buat Repository Baru

Buat folder baru, lalu masuk ke folder tersebut dan tuliskan
git init
pada folder tersebut untuk membuat repository

Periksa Repository

Buat copy dari repository:

  1. untuk lokal
    git clone /path/to/repository
  2. untuk non lolkal (remote server)
    git clone username@host:/path/to/repository

Alur

Terdapat 3 bagian pokok pada repository lokal, yang biasa disebut “trees”, yang pertama adalah Working Directory yaitu tempat menyimpan file - file yang asli, yang kedua adalah Index yang berguna untuk mengolah data dan yang terahir adalah Head yang berguna untuk menunjukkan aksi / commit terakhir yang dibuat.

Add & Commit

  1. Untuk menambah file yang akan dimasukkan ke dalam git,ketikkan
    git add <filename> atau git add *
  2. Setelah itu untuk commit pada aksi yang baru saja dilakukan ketikkan
    git commit -m"deskripsi yang akan dimasukkan"
    Artinya sekarang deskripsi telah masuk pada bagian Head.

Mengirim Perubahan

ketikkan git remote add origin <server> untuk mengkloning repository local ke server.

Untuk mengirim perubahan dari lokal ke server ketikkan
git push origin master

Branch

Pada git dimungkinkan untuk membuat beberapa kemungkinan, tanpa mengganggu repository utama

  1. Untuk membuat branch baru git checkout -b feature_x
  2. Untuk kembali ke repository utama git checkout master
  3. Untuk menghapus branch git branch -d feature_x
  4. Untuk menggunakan branch yang dibuat maka harus dikirim ke server dengan mengetikan
    git push origin <branch>

Update & Merge

  1. Untuk memperbaharui repositori lokal ketikkan
    git pull
  2. Untuk menggabungkan branch ke repository utama ketikkan
    git merge <branch>
  3. Terkadang proses penggabungan branch gagal, sehingga harus disunting/edit sendiri pada page yang
    ditunjukkan oleh gitu, setelah itu file baru yang dibuat harus di add dengan mengetikkan
    git add <filename>
  4. Sebelum branch digabungkan bisa juga untuk dilihat terlebih dahulu dengan mengetikkan
    git diff <source_branch> <target_branch>

Tagging / Menandai

Sangat dianjurkan membuat penanda atau tags untuk software yang dirilis.
Untuk membuat penanda baru dengan nama 1.0.0 ketikkan

git tag 1.0.0 1b2e1d63ff

1b2e1d63ff adalah 10 karakter pertama dari identitas komit yang referensikan ke penanda.

Log

  1. Untuk mempelajari history repository ketikkan
    git log
  2. Log yang ingin ditampilkan dapat disesuaikan dengan kebutuhan, mislanya untuk melihat commit dari author tertentu :
    git log --author=nama_author
  3. Untuk melihat log yang dipadatkan , satu baris per komit:
    git log --pretty=oneline
  4. Untuk melihat seluruhc ASCII branch serta nama authir dan taggnya:
    git log --graph --oneline --decorate --all
  5. Untuk Melihat hanya file yang dirubah
    git log --name-status
  6. Untuk melihat fitur lengkap lainnya
    git log --help

Menimpa Perubahan Repository Lokal

  1. Jika terjadi kesalahan maka dapat dikembalikkan ke posisi Repository terakhir yang ada di HEAD lokal,
    Perubahan dan filebaru yang telah ditambahkan ke indeks akan tetap tersimpan.
    git checkout -- <filename>

  2. Jika ingin membuang semua perubahan dan menggunakan Commint di lokal, ambil history terakhir dari server dan arahkan ke branc master lokal seperti ini

  • git fetch origin
  • git reset --hard origin/master

Hint Tambahan

  1. Built-in git GUI
    gitk
  2. Menggunakan output git penuh warna
    git config color.ui true
  3. Menunjukkan log satu baris per komit
    git config format.pretty oneline
  4. Menggunakan penambahan interaktif
    git add -i
